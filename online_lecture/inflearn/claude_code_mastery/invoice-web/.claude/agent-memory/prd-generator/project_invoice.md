노션 데이터베이스를 데이터 소스로 사용해 별도 DB 없이 견적서를 클라이언트에게 URL로 공유하는 서비스. PRD 작성 완료 (2026-04-21).

**핵심 아키텍처 결정:**
- 별도 DB 없음 — 노션이 유일한 데이터 소스
- 라우트: `/invoice/[notionPageId]` — 노션 페이지 ID가 곧 견적서 ID
- Server Component에서만 노션 API 호출 (`NOTION_API_KEY` 보호)
- PDF: MVP는 `window.print()` 방식으로 시작

**가정한 사항 (미결):**
- 품목 데이터: 노션 페이지 내 하위 테이블 블록 방식 우선 채택
- 부가세: 10% 자동 계산
- 공급자 정보: 환경변수 또는 노션 DB 포함 여부 미결정

**Why:** 노션 워크플로우를 유지하면서 클라이언트 경험만 개선하는 최소 변경 접근법.

**How to apply:** 향후 기능 추가 제안 시 "노션 중심" 원칙을 유지할 것. 별도 DB 도입은 범위 밖.
