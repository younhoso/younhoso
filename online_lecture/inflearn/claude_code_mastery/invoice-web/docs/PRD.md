# 노션 기반 견적서 공유 서비스 PRD

> 작성일: 2026-04-21 | 버전: v1.0 | 상태: Draft

## 1. 한 줄 요약

노션 데이터베이스에 입력한 견적서를 고유 URL로 클라이언트에게 공유하고, 클라이언트는 웹 브라우저에서 확인 및 PDF 다운로드할 수 있는 서비스

---

## 2. 문제 정의

- **대상 사용자**: 프리랜서 또는 소규모 에이전시 운영자 (노션을 이미 사용 중인 1인 개발자/디자이너)
- **해결하는 문제**: 노션에서 작성한 견적 내용을 클라이언트에게 보낼 때, 노션 공유 링크는 UI가 복잡하고 PDF 변환이 번거로움
- **기존 대안의 한계**:
  - 노션 공유 링크: 클라이언트가 노션 UI에 혼란을 겪음, 브랜딩 불가
  - Google Docs: 별도 작성 툴 필요, 노션 워크플로우 이탈
  - 전문 견적 SaaS(e.g. Freshbooks): 과금, 과도한 기능, 학습 비용

---

## 3. MVP 범위

### 포함

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 노션 DB 연동 | Notion API로 견적서 데이터 읽기 | P0 |
| 견적서 공개 페이지 | 고유 URL(`/invoice/[id]`)로 견적서 렌더링 | P0 |
| PDF 다운로드 | 브라우저에서 견적서를 PDF로 저장 | P0 |
| 견적서 항목 표시 | 품목명, 수량, 단가, 금액, 합계 표시 | P0 |
| 견적서 메타 표시 | 발행일, 유효기간, 공급자 정보, 클라이언트명 | P1 |
| 로딩/에러 상태 | 잘못된 ID 접근 시 404, 노션 오류 시 안내 메시지 | P1 |

### 제외 (나중에)

- **클라이언트 서명/승인 기능**: MVP 이후 — 법적 효력이 필요한 단계에서 추가
- **이메일 발송**: MVP 이후 — 현재는 URL을 직접 복사해서 전달
- **견적서 작성 UI**: 노션을 그대로 사용하므로 불필요
- **다국어 지원**: MVP 이후
- **견적서 목록/대시보드**: MVP 이후 — 노션에서 관리
- **열람 횟수/분석**: MVP 이후

## 4. 핵심 사용자 흐름

**흐름 1: 관리자가 견적서 공유 URL을 생성한다**

1. 관리자가 노션 데이터베이스에 견적서 항목을 입력한다
2. 노션 페이지 ID를 복사한다
3. `https://[서비스도메인]/invoice/[notionPageId]` 형태의 URL을 클라이언트에게 전달한다

**흐름 2: 클라이언트가 견적서를 확인하고 PDF를 저장한다**

1. 클라이언트가 URL을 브라우저에서 연다
2. 서버가 노션 API로 해당 페이지 데이터를 가져와 렌더링한다
3. 클라이언트가 견적서 내용을 확인한다
4. "PDF 다운로드" 버튼을 클릭하면 브라우저 인쇄 기능 또는 html2pdf로 PDF가 저장된다

## 5. 기능 명세

### 5-1. 노션 DB 연동

- **목적**: 노션을 데이터 소스로 사용해 별도 DB 없이 견적서 데이터를 관리한다
- **동작**:
  - `@notionhq/client` SDK로 노션 Page를 조회한다
  - 환경변수 `NOTION_API_KEY`로 인증한다
  - Page ID는 URL 파라미터(`[id]`)에서 추출한다
  - 노션 페이지의 Properties(데이터베이스 속성)에서 견적서 필드를 매핑한다
- **노션 데이터베이스 필드 정의**:

  | 노션 필드명 | 타입 | 용도 |
  |------------|------|------|
  | `title` | Title | 견적서 제목 |
  | `client_name` | Text | 클라이언트명 |
  | `issue_date` | Date | 발행일 |
  | `valid_until` | Date | 견적 유효기간 |
  | `items` | (하위 페이지 또는 멀티셀렉트) | 품목 목록 |
  | `note` | Text | 비고 |
  | `status` | Select | `draft` / `sent` / `accepted` |

- **예외 처리**:
  - 노션 API 오류(401, 404): 각각 "권한 없음", "견적서를 찾을 수 없습니다" 메시지 표시
  - 네트워크 오류: "잠시 후 다시 시도해주세요" 표시
- **완료 기준**: 노션 페이지 ID를 URL에 입력하면 해당 페이지의 Properties 데이터가 정상 파싱된다

---

### 5-2. 견적서 공개 페이지

- **목적**: 클라이언트가 별도 계정 없이 URL만으로 견적서를 열람할 수 있다
- **동작**:
  - 라우트: `src/app/invoice/[id]/page.tsx`
  - Server Component에서 노션 API 호출 (Next.js 15 fetch 캐싱 활용)
  - 견적서 레이아웃: 헤더(공급자 정보) → 클라이언트 정보 → 품목 테이블 → 합계 → 비고
  - 품목 테이블 컬럼: 번호, 품목명, 수량, 단가, 금액
  - 합계 행: 소계, 부가세(10%), 총액
  - 반응형: 모바일에서도 가로 스크롤 없이 읽기 가능
- **예외 처리**:
  - 존재하지 않는 ID: `notFound()` 호출 → 404 페이지
  - `status === 'draft'`인 경우: "준비 중인 견적서입니다" 표시 (공개 차단)
- **완료 기준**: 유효한 노션 Page ID로 접근 시 견적서가 정상 렌더링되고, draft 상태는 공개되지 않는다

---

### 5-3. PDF 다운로드

- **목적**: 클라이언트가 견적서를 PDF 파일로 저장해 보관하거나 출력할 수 있다
- **동작**:
  - "PDF 다운로드" 버튼 클릭 시 `window.print()` 트리거
  - `@media print` CSS로 헤더/푸터/버튼 숨김, 견적서 본문만 출력
  - 파일명 기본값: `견적서_[클라이언트명]_[발행일].pdf`
  - 대안: `html2canvas` + `jsPDF` 조합으로 버튼 클릭 즉시 PDF 생성 (브라우저 인쇄 팝업 없이)
- **가정**: MVP에서는 `window.print()` 방식으로 시작, 사용성 문제가 있으면 html2canvas 전환
- **예외 처리**: 브라우저 인쇄 취소 시 별도 처리 없음
- **완료 기준**: 버튼 클릭 시 견적서 내용이 PDF로 저장되고, 불필요한 UI 요소(네비게이션 등)가 제외된다

## 6. 데이터 모델 (핵심만)

별도 DB 없음. 모든 데이터는 노션에서 실시간 조회한다.

```typescript
// 노션 → 앱 내부 타입 매핑
type InvoiceItem = {
  id: string;
  name: string;       // 품목명
  quantity: number;   // 수량
  unitPrice: number;  // 단가 (원)
  amount: number;     // 금액 = 수량 × 단가
};

type Invoice = {
  id: string;           // 노션 페이지 ID
  title: string;        // 견적서 제목
  clientName: string;   // 클라이언트명
  issueDate: string;    // 발행일 (YYYY-MM-DD)
  validUntil: string;   // 유효기간 (YYYY-MM-DD)
  items: InvoiceItem[];
  subtotal: number;     // 소계
  tax: number;          // 부가세 (10%)
  total: number;        // 총액
  note: string;         // 비고
  status: 'draft' | 'sent' | 'accepted';
};
```

## 7. 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| Framework | Next.js 15 (App Router) | 현재 프로젝트 기반, Server Component로 API 키 보호 |
| UI | shadcn/ui + Tailwind CSS | 현재 프로젝트 기반 |
| 노션 연동 | `@notionhq/client` | 공식 SDK, 타입 지원 |
| PDF | `window.print()` → `html2canvas` + `jsPDF` | 의존성 최소화 우선 |
| 환경변수 | `.env.local` | `NOTION_API_KEY` 서버 사이드 보호 |
| 배포 | Vercel | Next.js 최적 환경, 무료 플랜으로 시작 |

**추가 설치 패키지:**
```bash
npm install @notionhq/client
# PDF 고급 옵션 필요 시:
npm install html2canvas jspdf
```

## 8. 비기능 요건

- **성능**: 견적서 페이지 로드 3초 이내 (노션 API 응답 포함). Next.js `revalidate` 60초로 캐싱
- **보안**: `NOTION_API_KEY`는 서버 사이드에서만 사용, 클라이언트 번들에 노출 금지. Server Component에서만 노션 API 호출
- **브라우저**: Chrome, Safari, Firefox 최신 버전. 모바일 Chrome/Safari
- **접근성**: URL을 아는 사람은 누구나 열람 가능 (별도 인증 없음) — `draft` 상태로 공개 제어
